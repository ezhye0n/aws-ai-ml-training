## 실습: AWS SageMaker AI 환경 구축 및 권한 설정

AWS 세이지메이커 AI(SageMaker AI)를 배포하고, 3일간의 실습에 문제가 없도록 **보안(MFA) 및 IAM 권한(Role)을 설정하는 전체 워크플로우 정리**

## 🔐 1. 2차 인증(MFA) 설정 및 로그인 관리

보안 규정 및 실습 리소스 보호를 위해 **2차 인증(MFA)** 설정이 필수적입니다. 공용 PC 사용 시 데이터 휘발에 주의해야 합니다.

### 💻 PC 환경별 분기 처리

- **개인 노트북 사용자:** [패스키 또는 보안 키] 옵션 선택 후 등록.
- **실습실 공용 PC 사용자:** PC 종료 시 초기화되므로 반드시 **개인 핸드폰**을 활용해 등록.
    1. 스마트폰에 **Google Authenticator (구글 OTP)** 앱 설치.
    2. MFA 디바이스 옵션에서 [인증 관리자 앱] 선택 후 화면의 QR 코드 스캔하여 등록.

> ⚠️ **중요 (AWS 버그 방지 팁):** MFA 등록이 완료되면 브라우저 세션에 즉시 반영되지 않고 경보가 뜰 수 있습니다. 반드시 **AWS 계정을 로그아웃하고 브라우저를 완전히 종료한 뒤, 30초 후에 재시작하여 다시 로그인**하세요.
> 

## 🚀 2. Amazon SageMaker AI 도메인 생성

실습의 중심이 되는 ML 플랫폼 인프라인 도메인을 생성하는 단계입니다.

1. AWS 콘솔 검색창에 **`아마존 세이지 메이커 AI` (뒤에 AI가 붙은 최신 서비스명 확인)** 입력 후 이동.
2. 좌측 메뉴에서 **[도메인(Domains)]** 클릭.
3. **[도메인 생성]** ➡️ **[단일 사용자용 설정(Quick Setup)]** 선택.
    - *주의:* 다수의 인원이 동시에 요청하면 AWS 가드레일에 의해 생성 거부(에러)가 날 수 있으므로 순차적으로 진행.
4. 생성이 완료되면 **[스튜디오 열기(Open Studio)]** 버튼 활성화 확인.

### 📌 고유 리소스 식별 및 자산 관리 (태그 설정)

빠른 설정으로 도메인을 생성하면 이름이 `default-2026xxxx...`와 같이 임의의 숫자로 자동 지정됩니다. 3일간 본인 리소스를 식별하기 위해 정보를 기록하고 태그를 부여합니다.

- **리소스 이름 기록:** 스튜디오 접속 후 우측 상단 프로필 아이콘을 눌러 나오는 고유 도메인 ID(예: `2026...`)를 별도 메모장에 백업.
- **태그(Tag) 설정:** 해당 도메인 상세 페이지의 [태그] 탭 ➡️ [편집]에서 아래 항목 입력 후 저장 (기업 실무 필수 프로세스).
    - **Key:** `user` / **Value:** 본인의 AWS 로그인 ID
    - **Key:** `team` / **Value:** `B` (영문 대문자)

## 🔒 3. IAM 역할(Role) 권한 고도화 (핵심)

도메인이 생성될 때 생성되는 기본 실행 역할(`AmazonSageMaker-ExecutionRole-XXXXXXXX`)은 최소 권한만 가지고 있으므로, 향후 S3 및 AI 서비스들과의 연동 실습을 위해 **5가지 필수 정책(Policy)을 추가**해야 합니다.

### 🛠️ 권한 추가 절차

1. AWS **IAM(Identity and Access Management)** 콘솔로 이동.
2. 좌측 메뉴 **[액세스 관리] ➡️ [역할(Roles)]** 클릭.
3. 검색창에 아까 백업해 둔 본인의 도메인 고유 숫자(2026...)를 입력하여 연결된 `ExecutionRole` 검색 후 클릭.
4. **[권한(Permissions)]** 탭 ➡️ **[권한 추가] ➡️ [정책 연결(Attach policies)]** 선택.
5. 아래의 **5가지 필수 AWS 관리형 정책**을 각각 검색하여 모두 체크 후 연결 완료.

### 📋 추가해야 할 5대 필수 정책 목록

- `AmazonS3FullAccess` (데이터셋 저장 및 로드를 위한 S3 완벽 권한)
- *기타 강의 교재에 명시된 필수 AI/ML API 접근 권한 4종* (Comprehend, SageMaker Full Access 등)
