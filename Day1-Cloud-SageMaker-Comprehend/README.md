## 📂 Day 1: AWS 글로벌 인프라 및 핵심 서비스 코어

### 🌐 1. AWS 글로벌 인프라스트럭처
* **리전 (Region):** 전 세계에서 서비스를 제공하기 위해 무리지어 있는 데이터 센터의 물리적 위치. 지연 시간(Latency), 비용, 법적 규제를 고려하여 선택.
* **가용 영역 (AZ, Availability Zone):** 하나의 리전 내에 위치한, 하나 이상의 데이터 센터로 구성된 격리된 그룹. 각 AZ는 독립된 전력, 쿨링, 네트워크를 갖추어 고가용성(HA)을 보장.

### 🛡️ 2. 네트워킹 & 컴퓨팅
* **VPC (Virtual Private Cloud):** AWS 계정 전용의 가상 네트워크 환경. 서브넷, 라우팅 테이블, 인터넷 게이트웨이를 통해 리소스 간의 격리 및 통신을 제어.
* **EC2 (Elastic Compute Cloud):** 안전하고 크기 조정이 가능한 컴퓨팅 파워를 클라우드에서 제공하는 가상 서버 인터페이스.

### 📦 3. 스토리지 서비스 비교
* **S3 (Simple Storage Service):** 확장성, 데이터 가용성, 보안 및 성능을 제공하는 객체(Object) 스토리지. 정적 웹 사이트 호스팅 및 데이터 백업에 탁월.
* **EBS (Elastic Block Store):** EC2 인스턴스를 위한 고성능 블록 스토리지 볼륨. 단일 EC2 인스턴스에 전용으로 연결되어 사용됨.
* **EFS (Elastic File System):** 여러 EC2 인스턴스에서 동시에 공유하여 액세스할 수 있는 확장 가능한 완전 관리형 파일 스토리지.

## 📂 Day 2: 머신러닝 모델 평가 및 AWS Comprehend 실습

### 💡 1. 머신러닝 핵심 개념: 과적합과 하이퍼파라미터
* **오버피팅 (Overfitting, 과적합):** 모델이 훈련(Training) 데이터에 너무 지나치게 맞춰져서, 새로운 데이터(Test 데이터)가 들어왔을 때 일반화 성능이 급격히 떨어지는 현상.
  * *해결 방법:* 학습 중간에 모델을 제어할 **검증(Validation) 데이터셋** 도입, 모델 복잡도를 낮추는 **규제화(Regularization)** 적용.
* **파라미터 vs 하이퍼파라미터:** 
  * *파라미터:* 가중치, 편향 등 모델이 학습을 통해 스스로 최적화하는 변수.
  * *하이퍼파라미터:* 학습률(Learning Rate), 에포크(Epoch) 등 **사용자가 직접 설정**하는 변수. (베이지안 최적화, 그리드 서치 등을 통해 최적 조합 탐색).

### 📊 2. 분류 모델 평가 지표: 혼동 행렬 (Confusion Matrix)
분류 모델의 예측 결과와 실제 정답 간의 관계를 정리한 2x2 행렬입니다.

| | 실제 Positive (1) | 실제 Negative (0) |
|---|---|---|
| **예측 Positive (1)** | **TP** (True Positive) | **FP** (False Positive) <br> *양치기 소년 오류* |
| **예측 Negative (0)** | **FN** (False Negative) <br> *놓친 오류 (의료/불량 진단에 치명적)* | **TN** (True Negative) |

* **정확도 (Accuracy):** 전체 중 맞춘 비율. 데이터 불균형이 심할 때는 신뢰도가 떨어짐.
* **정밀도 (Precision):** 모델이 1이라고 한 것 중 진짜 1인 비율 ($\text{FP}$ 통제가 중요할 때 사용).
* **재현율 (Recall):** 실제 1인 것 중 모델이 맞춘 비율 ($\text{FN}$ 통제가 중요할 때 사용).
* **F1 스코어 (F1-Score):** 정밀도와 재현율의 **조합평균(하모닉 평균)**으로 지표 간 균형을 평가.
  $$F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$
* **AUC (Area Under the Curve):** ROC 곡선 아래 면적으로, 1에 가까울수록 분류 성능이 우수함을 의미.

### 🛠️ 3. AWS Comprehend를 활용한 텍스트 분석
* **개요:** 사전 학습된 대규모 파운데이션 모델 기반의 **비정형 텍스트 분석 및 자연어 처리(NLP) 서비스**. API 호출만으로 고도화된 텍스트 인사이트 추출 가능.
* **핵심 기능:**
  1. **언어 감지 (`DetectDominantLanguage`):** 본문 언어 식별 및 신뢰도 점수 반환.
  2. **개체명 인식 (Entities):** 인명, 지명, 날짜, 기관명 등의 주요 키워드 구별.
  3. **감성 분석 (`DetectSentiment`):** 긍정, 부정, 중립, 혼합 확률 분석.
  4. **개인정보 감지 및 마스킹 (PII):** 이메일, 전화번호 등 민감 정보 위치 추적.
     * *개발 팁:* 마스킹 치환 시 문자열 길이 변화로 인덱스가 깨지는 것을 방지하기 위해 **역순(`Reverse=True`) 치환 알고리즘** 적용 권장.

### 🏢 비즈니스 응용 시나리오
* **글로벌 CX 모니터링:** 해외 고객 리뷰 및 피드백을 실시간 감성 분석하여 리스크가 높은 '부정 리뷰' 유입 시 즉각적인 대응 프로세스 가동.
* **콜센터 QA 및 개인정보 보호:** 고객 상담 스크립트에서 불만 유형을 개체명으로 분류하는 동시에, 로그 저장 및 공유 시 PII 감지 기능을 연동해 민감 정보를 자동 마스킹 처리하여 보안 컴플라이언스 준수.
